# MEDIOEVO / OSIT - Publicacion, fronteras y patrimonio

**Autor:** Luis René González López / Tren  
**Proyecto:** MEDIOEVO / OSIT / Observacionismo / Claudio  
**Version:** v0.1.0  
**Estado:** frontera publica inicial

## Proposito

Este documento define que se libera publicamente, que se mantiene privado y que requiere revision antes de publicarse.

La meta es doble:

1. Dar open source real: herramientas, principios, protocolos, schemas, ejemplos y documentacion util.
2. Proteger el patrimonio creativo, tecnico y comercial: libros, RPG/TCG, DUAT/GEODIA completo, Wabi-Sabi, Claudio privado, prompts crudos, datasets, secretos y runtime interno.

## Capa abierta

Puede publicarse como open source o documentacion publica:

- Observacionismo como metodo operativo.
- PSI / OSIT para IA con receptor con estado.
- R, Phi_eff y J_c como variables operativas.
- Segunda Perdida y handoff.
- ActionGate, GhostGate, WitnessLog, claim boundaries y source cards.
- CEREBRO como biblioteca humana de navegacion.
- Herramientas pequenas: handoff generator, witness log, claim classifier, canon compiler, source-card generator, secret scanner, context compressor, state estimator.
- Demos sinteticas y tests reproducibles sin datos privados.
- Benchmarks propuestos, siempre con alcance claro.

## Capa semiabierta

Puede publicarse despues de revision:

- UI mockups de Claudio / Argus sin endpoints sensibles.
- Geodia city shell read-only.
- ObservaScript / OP parser minimo.
- OML-RS core si no contiene rutas privadas, secretos ni claims subjetivos.
- Papers de trabajo con estado epistemico explicito.

## Capa privada protegida

No se publica como open source:

- Los 35 libros completos, borradores, arcos no publicados y biblia narrativa.
- RPG, TCG completo, mecanicas privadas, assets, reglas internas y economias.
- DUAT / GEODIA completo, kernel, ISO, backend, runtime y assets privados.
- Wabi-Sabi completo, router real, policies, prompts maestros, memoria y seleccion de agentes.
- Claudio core privado, datos de usuario, pipelines comerciales, automatizaciones sensibles.
- MOI / sistema de investigacion Claudio completo, conectores, fuentes privilegiadas y prompts premium.
- Secrets, tokens, claves, `.env`, logs privados, datos personales, cuentas y despliegues.

## Reglas clean-room

Material filtrado, source maps o leaks:

- No copiar codigo literal.
- No integrar funciones de evasion, ocultamiento, bypass, git-blame fake o metadata scrubber para ocultar autoria.
- Extraer solo patrones generales permitidos.
- Reescribir implementaciones desde cero.
- Documentar origen como referencia no confiable.
- Revisar licencia antes de publicar.

## Estados epistemicos

Toda publicacion debe marcar afirmaciones como:

- CERTEZA: evidencia directa, test, hash, logica cerrada o fuente visible.
- INFERENCIA: plausible, pero no demostrado.
- INCOGNITA: falta dato, ejecucion o validacion.
- BLOQUEO: riesgo tecnico concreto, secretos, deploy, borrado, datos privados o accion no autorizada.

## Frase de frontera publica

> MEDIOEVO libera fundamentos, herramientas y metodos para agentes con evidencia, continuidad y gates. El universo narrativo, los sistemas completos y el runtime privado permanecen protegidos.
